### 1.2 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| **开发语言** | C# (.NET 8) | 性能优秀，类型安全，跨版本兼容性好 |
| **UI框架** | WPF + MaterialDesignInXamlToolkit | 现代化界面，成熟稳定 |
| **架构模式** | MVVM | 代码结构清晰，易于维护测试 |
| **日志框架** | Serilog | 结构化日志，支持多种输出目标 |
| **部署环境** | WinPE (Windows ADK) | 官方支持，驱动完整，稳定性高 |
| **镜像工具** | DISM | Windows 原生工具，功能完善 |

### 2.2 配置传递机制

```
当前系统（工具运行环境）
    │
    ├── 生成 unattend.xml ───────┐
    ├── 备份驱动 ────────────────┤
    ├── 配置 RustDesk ───────────┤───▶ 保存到目标磁盘临时目录
    ├── 备份 IP 配置 ────────────┤      (D:\WinDeployTemp\)
    └── 准备 WinPE ──────────────┘
                │
                ▼
            重启进入 PE
                │
                ▼
WinPE 环境
    │
    ├── 应用镜像到目标分区
    ├── 将 unattend.xml 复制到 C:\Windows\Panther\
    ├── 将驱动复制到 C:\Drivers\
    ├── 将 RustDesk 安装包和配置复制到 C:\Temp\
    └── 配置 BCD 引导
                │
                ▼
            重启进入新系统
                │
                ▼
新系统首次启动
    │
    ├── Windows Setup 读取 unattend.xml
    ├── 自动设置语言、区域、用户名
    ├── 执行 FirstLogonCommands:
    │       ├── 安装 RustDesk (msiexec /i rustdesk.msi /quiet)
    │       ├── 恢复 IP 配置 (netsh -f ip_backup.txt)
    │       ├── 安装驱动 (pnputil /add-driver C:\Drivers\*.inf /subdirs /install)
    │       └── 清理临时文件
    └── 自动登录桌面
                │
                ▼
            RustDesk 已连接，可远程操作
```

## 三、关键模块设计

### 3.1 系统检测模块

**功能**：
- 检测当前 Windows 版本
- 检测 BIOS 模式（UEFI/Legacy）
- 检测磁盘信息（分区表类型、分区列表）
- 检测 Secure Boot 状态
- 检测 TPM 状态

**输出**：
- 兼容性报告（是否支持目标系统）
- 警告提示（如 Secure Boot 未开启但目标系统需要）

### 3.2 镜像管理模块

**支持格式**：
- ISO：挂载后提取 install.wim/install.esd
- WIM：直接使用 DISM 操作
- ESD：DISM 支持，但需转换为 WIM 才能修改（可选）

**功能**：
- 镜像文件校验（SHA256）
- 镜像信息读取（版本、架构、索引）
- 镜像预览（显示包含的版本列表）

### 3.3 驱动管理模块

**驱动识别**：
- 使用 DISM 枚举已安装驱动
- 过滤掉微软内置驱动（OEM 驱动保留）
- 显示驱动名称、版本、厂商、设备类型

**驱动备份**：
- 默认不勾选
- 用户可逐个选择要备份的驱动
- 导出为压缩包，保存到日志目录

**驱动恢复**：
- 在新系统首次启动时通过 pnputil 安装
- 只安装备份时勾选的驱动

### 3.4 配置生成模块

**unattend.xml 生成**：
- 根据用户选择的配置动态生成
- 包含以下阶段：
  - `windowsPE`：磁盘分区、镜像选择
  - `specialize`：计算机名、时区
  - `oobeSystem`：用户账户、自动登录、FirstLogonCommands

**RustDesk 配置**：
- 生成 RustDesk 客户端配置文件
- 包含中继服务器地址、密钥、密码设置

**IP 配置备份**：
- 使用 `netsh interface ip dump` 导出
- 保存为可执行的脚本文件

### 3.5 部署服务模块

**阶段1：准备阶段**
- 验证所有配置是否完整
- 生成所有配置文件
- 准备 WinPE 引导（复制 boot.wim，配置 BCD）
- 显示确认对话框
- 重启进入 PE

**阶段2：PE 部署阶段**
- 启动后挂载目标磁盘
- 执行磁盘分区（删除系统分区，保留数据分区）
- 使用 DISM 应用镜像
- 注入驱动到离线系统
- 复制配置文件到离线系统
- 配置 BCD 引导
- 重启

**阶段3：验证阶段**
- 等待新系统启动完成
- 验证 RustDesk 连接状态
- 显示部署结果

## 四、日志系统设计

### 4.1 日志存储位置

```

│   ├── 2025-02-08_091530\

### 4.2 日志文件组织

```

├── Deploy.log                        # 主日志（合并所有阶段）

├── Phase2_WinPE.log                  # PE 阶段详细日志

├── Phase4_Validate.log               # 验证阶段日志
├── Result.json                       # 部署结果摘要

    └── After.json
```

### 4.3 日志级别

| 级别 | 用途 |
|------|------|
| Verbose | 最详细的调试信息 |
| Debug | 调试信息，包含变量值 |
| Information | 正常流程的关键节点 |
| Warning | 警告，不影响流程但需注意 |
| Error | 错误，可能影响部署结果 |
| Fatal | 致命错误，部署无法继续 |

### 4.4 日志格式

```
[2025-01-12 14:30:22.345] [INFO] [Phase1] [DiskService] 开始备份驱动

## 五、错误处理设计

### 5.1 错误分类

| 类别 | 说明 | 处理方式 |
|------|------|---------|
| 可恢复错误 | 单个操作失败，不影响整体流程 | 记录警告，继续执行 |
| 可重试错误 | 临时失败，可重试 | 重试 3 次，失败后记录错误 |
| 致命错误 | 核心步骤失败，无法继续 | 停止部署，记录错误，提示用户 |

### 5.2 错误码设计

| 错误码 | 说明 |
|--------|------|
| 0x1001 | 镜像文件校验失败 |
| 0x1002 | 磁盘分区失败 |
| 0x1003 | DISM 应用镜像失败 |
| 0x1004 | 驱动注入失败 |
| 0x1005 | BCD 配置失败 |
| 0x1006 | 无法进入 WinPE |
| 0x2001 | RustDesk 安装失败 |
| 0x2002 | IP 配置恢复失败 |
| 0x2003 | 驱动安装失败 |

### 5.3 用户提示

- 错误发生时，界面显示友好的错误说明
- 提供"查看日志"按钮，直接打开对应日志文件
- 提供"复制错误信息"功能，方便反馈问题

## 六、安全与隐私

### 6.1 敏感信息处理

| 信息类型 | 处理方式 |
|---------|---------|
| RustDesk 密码 | 日志中显示为 `********` |
| IP 地址 | 完整记录（网络配置需要） |
| 计算机名 | 完整记录 |
| 用户名 | 完整记录 |

### 6.2 权限要求

- 工具需要以管理员权限运行
- WinPE 阶段自动以 SYSTEM 权限执行
- 日志文件设置合适的 ACL，防止未授权访问

## 七、兼容性设计

### 7.1 Windows 版本兼容性

| 功能 | Win7 | Win8/8.1 | Win10 | Win11 | Server 2016/2019 |
|------|------|----------|-------|-------|------------------|
| 作为源系统运行工具 | 部分支持 | 支持 | 完全支持 | 完全支持 | 支持 |
| 作为目标系统部署 | 支持 | 支持 | 支持 | 支持 | 支持 |
| DISM 高级功能 | 受限 | 基本 | 完整 | 完整 | 完整 |
| PowerShell 脚本 | 2.0 | 3.0/4.0 | 5.1+ | 5.1+ | 5.1+ |

### 7.2 特殊处理

**Win7 作为源系统**：
- 需要安装 .NET Framework 4.8
- 部分功能受限（如某些 DISM 参数）

**Win11 目标系统**：
- 自动添加 TPM/Secure Boot 绕过配置（如需要）
- 检测并提示 Secure Boot 状态

## 八、扩展性设计

### 8.1 自定义脚本支持

工具提供"高级选项"，允许用户在以下阶段执行自定义脚本：

| 阶段 | 执行时机 | 环境 |
|------|---------|------|
| 部署前 | 重启进 PE 前 | 原 Windows 系统 |
| 部署中 | PE 中镜像应用后 | WinPE 环境 |
| 部署后 | 新系统首次登录时 | 新 Windows 系统 |

支持格式：`.bat`, `.cmd`, `.ps1`, `.exe`, `.msi`

### 8.2 插件机制（未来扩展）

预留插件接口，未来可支持：
- 额外的远程控制软件（TeamViewer、AnyDesk 等）
- 系统优化脚本
- 软件预装包
